# 雑用担当メモ

## 担当範囲
他の担当（リーダー／実装担当／コンテンツ・デザイン担当）に明確に当てはまらない作業すべて。

例:
- git 操作の代行（実装担当が手一杯のとき）
- 環境確認（Node.js のバージョン確認、ツール導入確認など）
- ファイル整理・リネーム
- ツール・MCP サーバーの設定確認
- 公開情報の調査
- ドキュメント整理

## 作業ルール
- リーダーから振られた作業は、まず内容を整理してから着手すること
- 結果はリーダーに報告できる形で簡潔にまとめること
- 他担当の領域に踏み込まないこと（迷ったらリーダーに確認）
- 環境・ツール周りで変更を加えた場合は、その都度このファイルに記録すること

## マージ後の定例フロー
- PR が main にマージされたら、`main` へ切替 → `git pull` で最新化 → マージ済み feature ブランチをローカル・リモートとも削除 → 次の作業は最新 main から新しい feature ブランチを作成して開始する。
