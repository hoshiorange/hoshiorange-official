# コンテンツ・デザイン担当メモ

## サイトの目的
**hoshiorange の公式ハブサイト**。各 SNS / 活動拠点へのリンクを集約し、SNSから流入した人が他の活動にも回遊できる「中継地点」として機能させる。

- 想定読者: 各 SNS（X / YouTube 等）から「hoshiorange って何者？」と興味を持って訪れた人
- 役割:
  1. 各種 SNS アカウント・チャンネル・ブログ等への導線
  2. プロフィール／自己紹介
  3. お仕事募集・連絡先
  4. 将来作る HP・ブログ・note 等を載せる場所

似た系統: Linktree / Pop / lit.link を、自分のドメインで・自分の見た目で持つイメージ。

## 表示名
- **ほし**（サイト上の名乗り）
- 英表記の hoshiorange はサブ／URL／OGP 等で使用

## コンテンツ構成案（初期）
方向性: **こだわりありのHP風**（Linktree的な縦リスト一辺倒ではなく、セクション分けして「ちゃんとしたサイト」感を出す）

1. **ヒーロー** — 公式ハブ感（Official Hub）を主役に。アイコン＋表示名「ほし」＋リード文

   - 現状の Hero テキスト要素（変更前）:
     - `badge`（"Official Hub"）… ハードコード。**気に入っているので残す/主役に昇格**。
     - `title`（h1）… `profile.tagline` 相当の「夜空に届け、ほしのつぶやき。」をハードコードで2行表示。**削除（ポエム調キャッチ）**。
     - `sub`（`profile.subTagline`「ゲーム、創作、コード。三つの軌道を行き来する個人ハブ。」）… 星・軌道の比喩を含むため About 方針（比喩排除）と不整合。**削除 or 差し替え**。
     - `lead`（`profile.heroLead`「ほしの活動拠点です。…」）… 旧 About 統合のリード文。**残す**。
   - **採用案: 案A（英語ラベル主役）**

     - h1（旧 title）: 「hoshiorange」＋表示名「ほし」を組み合わせたロゴ的タイトルに。例: 大きく `hoshiorange` ／小さく `Official Hub` ／表示名 `ほし`。ポエム文言は入れない。
     - sub: 削除（または heroLead に一本化）
     - lead: heroLead をそのまま使用
   - 文言の最終形は下の「Hero テキスト構成案」を参照。
2. **About** — 活動拠点であることを示すだけ（ミニマル）
   - **本文方針: ミニマル方針に決定（2026-06-20 ユーザー確定）／文面は検討中**

     - 三つの活動軸は pillars カード側で見せるので、本文では繰り返さない。

     - 方針が「活動拠点を示すだけ」になったため、About を独立セクションとして残すか Hero に統合するかも選択肢（後述）。
     - 旧文面: `src/data/profile.ts` の `aboutBody`。新3パターン案をユーザー確認中、確定後に差し替え予定。
     - **ミニマル本文 3パターン案（ユーザー確認中）**:
       - A: 「ほしの活動拠点です。X や YouTube など、いろんな場所での活動をここにまとめています。」

3. **Links（メイン）** — 各 SNS・チャンネル・サイトへのリンクカード
   - 初期掲載: X (Twitter), YouTube
   - 将来追加: ブログ、note、HP など（追加しやすい構造に）
4. **YouTube 最新動画** — チャンネルの最新数件をサムネ付きで表示
5. **X 最新ポスト** — 公式埋め込みタイムラインで最新を表示
6. **Contact / お仕事募集** — 連絡先・依頼の窓口

セクション順序は実装時に再検討（Hero → About → Links → 動的フィード(YouTube/X) → Contact が候補）。

## Hero テキスト構成案（2026-06-20）

前提: ポエム調キャッチ削除、「Official Hub」感を主役。骨格＝英語ラベル＋表示名「ほし」＋リード文。

### 案A（英語ラベル主役 / 推し）
- badge: `Official Hub`（現状維持）
- h1: `hoshiorange`（英字大）＋小さく `Official Hub`、サブで `ほし` を添える
  - 表示例: 大「hoshiorange」／小「Official Hub」、表示名は lead 内 or 小ラベルで「ほし」
- sub: 削除
- lead: `ほしの活動拠点です。X や YouTube など、いろんな場所での活動をここにまとめています。`
- 違い: 英語ラベルでスッと「公式ハブ」と伝わる。クール・無駄がない。

### 案B（日本語併記 / やわらかめ）
- badge: `Official Hub`（現状維持）
- h1: 大「ほし」＋小さく英字 `hoshiorange — Official Hub`
- sub: 削除
- lead: 同上 heroLead
- 違い: 表示名「ほし」を主役にし英語は補助。日本語の人に親しみやすい。

### profile.ts への影響（実装担当向け）
- `tagline`（'夜空に届け、ほしのつぶやき。'）… **削除**（Hero では未使用化、metadata 等で参照あれば差し替え要確認）
- `subTagline`（'ゲーム、創作、コード。三つの軌道…'）… **Hero から削除**（比喩を含み About 方針と不整合）。他で未使用なら定数自体も削除可
- `heroLead` … 維持
- `displayName: 'ほし'` / `handle: 'hoshiorange'` … h1 のラベル組み立てに使用

## 設計上の意識
- リンク追加・削除がしやすい構造にする（データを JSON/TS 配列で外出し）
- 「まだないけどそのうちやる」リンクも、コメントアウトや coming soon 表示で枠を確保しておけるように
- SEO 対策（OGP、title、description）→ SNS でシェアされたとき見栄えよく
- モバイルファースト（SNS から飛んでくる人はスマホが多い前提）

## ほしさんの立ち位置（サイトのトーンの土台）
複合型。以下を併せ持つ:
- ゲーム実況・配信者系
- クリエイター（イラスト/動画/音楽など）
- エンジニア/技術者

→ どれか一色に染めず、活動領域別にリンクをグループ化できる構造を意識する。

## デザイン方針
- **テーマ: 夜空・星空系**（「ほし」要素を活かす）
  - キーカラー: 紺・黒ベース＋オレンジ（hoshiorange の orange を差し色に）＋星の白
  - 背景に星のドット／グラデーションを散らす演出ありで検討
- **ダーク/ライト両対応＋手動トグル**
  - デフォルト: システム設定追従
  - トグルでユーザーが切替可能、選択は localStorage で永続化
  - 星空系なのでダークが主役、ライトは「朝焼け／昼の空」イメージで設計
- フォント: 日本語＋英語のバランスが取れるもの（Google Fonts から選定予定、未定）
- トーン: シンプル・清潔感を保ちつつ、星の演出で少し遊びを入れる
