# 010 こおりのぬけみち 実装計画（M0〜M2 ＋ フェーズ2）

## 概要
Laboratory 配下のゲーム「こおりのぬけみち」（氷床スライドパズル）の実装計画。
008 で用意した Coming Soon ページ（`/lab/koori-no-nukemichi`）を、段階的に実ゲームへ置き換えていく。
本ドキュメントはマイルストーン（M0 / M1 / M2）ごとの実施プランを定義する。実装本体は別エージェントが並行で進める。

## 確定仕様サマリ（grillMe で確定）

### ゲームルール
- 氷床スライドパズル。方向入力で滑走し、**壁・盤面の縁・通常床**で停止する。
- **ゴールで「止まれば」クリア**（通過は不可）。失敗（落下・ゲームオーバー）は無い。
- 機能: リセット / アンドゥ（1 手）/ 手数表示。アニメーション中は入力を無視する。

### タイル・データ表現
- タイル種別: `0`=氷床 / `1`=壁・岩 / `2`=通常床。スタート・ゴールは座標で管理。
- データ形式: スカラー列（`width`, `height`, `startX/Y`, `goalX/Y`）＋ `data` テキスト。
  - `data` は `"111,101,121"` 形式。**1 マス 1 文字・カンマ = 行区切り**。

### 進行・記録
- 2 階層構成: ワールド（章）→ ステージ。章内はフリー選択。章を全クリアで次章を解放。
- クリア記録は **localStorage**（フェーズ1。サーバ同期はフェーズ2）。

### アーキテクチャ
- **ゲームコア = 純 TS**（描画から分離）。方向入力に対する「操作手順＋手数」を返す純粋ロジック。
- **プレイヤー（描画）= Pixi.js v8**（Canvas, `ssr: false`）。
- **エディタ = DOM**（HTML/CSS の素の DOM 操作）。
- 将来スマホアプリ化する場合は**描画レイヤーのみ新規構築**できるよう、コアと描画を疎結合に保つ。

### データベース（M1 以降）
- Supabase Postgres。`world`（章）と `stage` の 2 表。`scores` はフェーズ2。
- RLS: 公開ステージは**誰でも read**・**書き込みは admin のみ**。

### 認証（M1 以降）
- **Google OAuth のみ** ＋ `admins` ホワイトリスト（管理人だけ）。UI 側でも admin 判定。
- 一般ユーザーにはログイン UI を見せない。

### エディタ（M1 以降）
- タイル塗り（ドラッグ可）/ スタート・ゴール配置 / リサイズ（リサイズ時は全クリア）。
- テストプレイ。**作成者が実際にクリアして初めて公開解放**（マリオメーカー方式・`author_moves` を保存）。
- 下書きと公開を分離。公開後は盤面ロック（メタ情報のみ編集可・複製は可）。章 CRUD。

### 画面・URL（案2 採用）
- **単一 URL の没入型ゲームアプリ**。内部状態で画面遷移（章選択 / ステージ選択 / プレイ / エディタ）。
- 全画面専用レイアウト = サイトの **Header / Footer は非表示**、**夜テーマ固定**。
- 共有はフェーズ2のボタン方式。`?s=` での直行受け口だけ先行して用意。
- 導線は **Laboratory カードのみ**（008 で設置済み）。

### アート・音
- ビジュアル: 夜 × 氷 × オーロラ。クリーンなイラスト調。ほしマスコット。
- 素材は GPT 生成（後日差し込み）。アセットマニフェストで差し替え可能にしておく。
- SFX: フェーズ1から入れる。**既定はミュート**。BGM は簡単なら同梱、難しければフェーズ2。

### 環境変数
- `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`（M1 以降）。
- `SUPABASE_SERVICE_ROLE_KEY` はフェーズ2。
- **env が無くてもローカルアダプタで動く**方針（M0 はオフラインで完結）。

## 実施プラン

### M0 ― 純 TS コア ＋ 最小 Pixi で 1 ステージ実プレイ（第1リリースの土台）
オフライン・env 不要で 1 ステージが実際に遊べる状態をゴールとする。

#### 純 TS ゲームコア
- [ ] タイル/盤面のデータ型を定義（`0/1/2` ＋ start/goal 座標、`data` テキストのパーサ）
- [ ] `data` テキスト（`"111,101,121"`）⇔ 盤面配列の相互変換ユーティリティ
- [ ] スライド移動ロジック（方向入力 → 壁・縁・通常床で停止する到達セル計算）
- [ ] ゴール判定（**ゴールで停止したらクリア**・通過はクリアにしない）
- [ ] 手数カウント / アンドゥ（1 手）/ リセットの状態管理
- [ ] 操作手順（入力列）と手数を返すインターフェース（フェーズ2 のサーバ再生検証を見据える）
- [ ] コアの単体検証（代表ステージで移動・停止・クリア・アンドゥが期待通り）

#### 土台スキャフォールド（疎結合の足場）
- [ ] リポジトリ（データ取得）インターフェースを定義（`world` / `stage` を返す抽象）
- [ ] ローカルアダプタ実装（ハードコードした 1〜数ステージを返す。env 不要）
- [ ] 認証スタブ（常に「未ログイン一般ユーザー」を返すダミー。M1 で本実装へ差し替え）
- [ ] アセットマニフェスト（画像・SFX のキー → パス対応表。後日 GPT 生成素材を差し込む口）

#### 最小 Pixi プレイヤー
- [ ] Pixi.js v8 を導入し、`ssr: false` の dynamic import で Canvas を描画
- [ ] 盤面（氷床/壁/通常床/スタート/ゴール）を最小グラフィックで描画
- [ ] キーボード方向入力 → コアに渡し、滑走アニメーションを再生（**アニメ中は入力無視**）
- [ ] 手数表示・リセット・アンドゥの最小 UI
- [ ] クリア表示（ゴールで停止時）

#### プレビュールート・検証
- [ ] `/lab/koori-no-nukemichi`（または配下）で最小プレイヤーを表示（Coming Soon は M2 で正式置換）
- [ ] 全画面専用レイアウト（Header/Footer 非表示・夜テーマ固定）の土台を用意
- [ ] `npx tsc --noEmit` 成功
- [ ] `npm run build` 成功（env 無しでもビルド・プレイ可能）
- [ ] dev で 1 ステージを通しプレイ（移動・停止・クリア・アンドゥ・リセット）
